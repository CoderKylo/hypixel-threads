Hear My Opinion out, Words of someone Daily Driving Linux

To Start Out use Debian based Distros that are beginner friendly eg Ubuntu POP_OS and Zorin OS etc, Now With Windows 11 Microsoft showed us how they dont care bout older hardware, TPM is useless unless even for big companies that Handle multi million transactions, even worser my local McDonnalds is still using VISTA Like WHY VISTA it was the worst! but back to the Topic, Steam deck will be responsible for huge amount of Linux game support because Valve will base it on Arch... why idk but al least we will be able to play Valorant and other EAC Games on These Awesome OS'es. Do you have a lot of RGB in your PC like i do? dont worry you dont need to rely on the manufactures software, we Have OpenTGB and RemoteLight to pair it with OpenRGB and it is awesome, may not work for those that use an Corsair Commander Pro instead of an Lightning node for LL/QL Series Fans but it can be easily tweaked using RemoteLight. Epic Games Launcher not Linux Compatible? Heroic Launcher is there with all of Epic's Functionality, redeeming free games lacks only... Minecraft works, but you will need to use Badlion or Lunar for best FPS, you could tweak OptiFine but now im just too lazy to even do it. Forge works fine on Linux too and Java will work anyways since is it not .exe and only needs JRE to be installed. EXT4 is superior to FAT32 and NTFS, Period. Cannot Overclock your GPU? there is an MSI Afterburner alternative, called "GWE" (GreenWithEnvy) need HardwareInfo 64? GTKStressTesting works out the same with temp monitoring core utilisation on visual "Pillars". Linux never BSOD's linux has a very stable and Hard kernel that nearly never Panics (Kernel Panic is a Linux Like BSOD). You cannot get viruses, this is never 100% true but Linux is 100% less likely to get viruses as every operation of deletion of Important System Files needs Authorization. You have tons of Troubleshooting Materials Online! its basically copy Paste to solve any problem. iPhone software support is very slim, needs to be worked on but leave that to me, an iBEC and iBSS expert.

Install Updates whenever you want, Linux will never nag you to do so.

Updates will never install Automatically as every Action that requires changing replacing and removing files will need your Authorization, Awesome right?

No Bloatware, Privacy at its Finest

Linux will never spy on you and collect user data, unless you opt in (Ubuntu) any other Distro that i know than Ubuntu will never even ask to collect your data because the People behind these OS'es Value Privacy as i do and you should too! OneDrive hogging CPU? Not even possible here, Linux comes 100% Free of additional software unless you chooise to do so (Ubuntu) and it actually comes with my Favorite browser, Firefox and not Crappy Edge, i hated Edge before but i hate chrome even more and Chromium based Edge is just cancer, For me.

Reliable, never spits Errors unless you messed up.

Above says enough, I guarantee you wont get an error exept you mess up.

Even a Potato can run Linux!

512mb/ 1GB RAM? No Problem! Use Kubuntu or an other Lightweight Ubuntu based Distro.

Final Verdict

It may be hard to get used to the Command line for nearly everything, but hey the Payoff is nice no viruses errors and gernerally Smooth Sailing Privacy like never Before an awesome Settings app on all Distros and since the Steam Deck Leaks Linux will get a huge boost in compatible software.

Interesting topic but quite a lot of info is heavily misinterpreted here or can be considered misleading or just inaccurate.
The_Hackintosh said:
even worser my local McDonnalds is still using VISTA Like WHY VISTA it was the worst!
Click to expand...
if it aint broke no need to fix it. A lot of companies in general that use old operating systems have hardware that is built for devices around that era. Mcdonalds is giant money hungry corporation so if there was a way to save money they'd prolly do it by now. It's most likely certain applications or hardware just won't properly work with newer systems so they don't bother updating or the software they use no longer receives support. It could also be that the old operating system is used in crucial system infrastructure which simply can't be taken down to be fixed, if it would be taken down mcdonalds could suffer serious earnings losses and most importantly lose value.

The_Hackintosh said:
Steam deck will be responsible for huge amount of Linux game support because Valve will base it on Arch... why idk but al least we will be able to play Valorant and other EAC Games on These Awesome OS'es.
Click to expand...
also not true, valve is just developing protondb which emulates windows apps in linux, the same thing wine does, just better. However protondb is exclusive to steam games. So valorant wont be supported unless riot games do something about it. Another issue is the anticheat, lots of games on steam that require an anticheat may not run or function correctly.

As for the reason they switched to using arch linux is because I quote

"So, Arch Linux, one of the main reasons, there's a couple, but the main reason is the rolling updates of Arch allows us to have more rapid development for SteamOS 3.0," says Yang. "We were making a bunch of updates and changes to specifically make sure that things work well for Steam deck, and Arch just ended up being a better choice for them."
-Valve designer, Lawrence Yang
Arch linux also has great documentation and most likely allowed valve to get really deep and modify the os to their liking.

If you have an amd gpu, amd's radeon graphic driver allows you to do quite a bit in house.

Click to expand...
Not true, this depends entirely on the user. If the user is unfamiliar with computers then they themselves will most likely download the virus and give it all the perms it needs. This is also how most people get viruses on windows. The different here is most viruses are .exe's so they won't run on linux but a sophisticated virus could find it's way onto linux machines.

The_Hackintosh said:

The_Hackintosh said:
Above says enough, I guarantee you wont get an error exept you mess up.
Click to expand...
Also not entirely true. You can have hardware fail and linux will report errors but you yourself didn't mess up, it's just faulty hardware or a faulty config. I've seen this first hand with a friend who had memory clocks set to auto in their asrock bios and linux and windows would both throw memory crashes. Issue was resolved by setting the memory to a specific clock speed. Or @Ezap (sorry for ping), faulty cpu. Errors are just a way for the system to report to you that something happened that it couldn't handle and showing you all the info it has on it.
