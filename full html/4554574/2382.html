Correct me if it is of my ignorance,

I know that Lunar has obfuscated logs and some other lore for being questionable, but most of it is dismissed. Still, Moonsworth LLC makes money off of cosmetics and other things alike. Would be cool to mess around with the code.

But I do understand completely that they wouldn't necessarily be comfortable have their code forked which I without a doubt know there would be tons of them fixing features Lunar hasn't fixed, or a arguably better Lunar Client too. You can put licensees on your code like GPL v3, to mitigate some of that.

Honestly, in regards to the developers and Moonsworth ultimately making money for their great work, and from me I don't see having forks be problem. That happening would only make Lunar more popular and more people using the forked versions, then more likely to buy stuff.

For my philosophy the details of the code and it's code for purchased cosmetics and giving them from the API would obviously need to be present.

What does anyone think? I am starting to get tired of mods being removed (freelook is fair I guess..) or disabled for months like auto-text. I would like to just be able to clone Lunar and make my changes to it and separate my agenda from the original devs. Not too much harm their, but yeah.

Because some developers don't want their code to be publicly visible for whichever reasons. It's like asking why your favourite games aren't open sourced, it's to help protect the rights to the code that the company owns. It's a good idea in practice but in reality very few people actually care. After all you don't want your work to be used in something else that generates more money. Open sourcing their client can also allow for easier injection of cheats into the client which is what custom clients are meant to prevent.

Good idea on paper but not so much in reality, there have also been multiple talks from a few people about starting an open source mc client but a lot of those people don't know basic java let alone oop and are just following a yt vids so not a very worthwhile project to contribute to.

On another note sometimes things are closed source to keep people away from abusing the code. For example I've got a few mods in the works but they can definitely be abused so keeping them closed source allows me as a developer to control who knows how this works and whatnot. This is most prominent in hacked clients as they close source themselves so other developers can't copy/patch their client's bypasses.
